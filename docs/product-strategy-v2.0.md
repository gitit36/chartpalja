# 차트팔자 — 제품 기획서 v2.0

v1.1 대비 변경 사항: A/B 비교 제거 → 단일 엔진(A) 확정, 100세 타임라인, 비교하기(궁합) 기능, 음력 윤달 지원, 엔진 v5 안정화, LLM 하이브리드 해설 시스템

---

## 1. 제품 철학

차트팔자는 **"내 인생에 구조가 있다는 감각"**을 파는 서비스다.

사람은 사주가 과학인지 아닌지 궁금한 게 아니다. **"지금 겪는 혼란이 일시적인지, 의미가 있는지"**가 궁금한 거다. 사람은 랜덤한 고통은 견디지 못하지만, 서사 안의 고통은 견딘다. 차트는 인생을 랜덤에서 서사로 전환시키는 장치다.

우리가 파는 것은 데이터가 아니라 **"당신의 인생에는 리듬이 있고, 지금은 그 리듬의 이 지점입니다"**라는 감각이다.

---

## 2. 타겟 사용자

25~39세, 커리어/연애/결혼에서 결정을 앞두고 있는 사람. 사주를 "믿는" 사람이 아니라 "궁금한" 사람. 운세 앱을 깔아봤지만 텍스트가 다 비슷해서 실망한 경험이 있는 사람.

## 4. 아키텍처

### 4-1. 점수 시스템: 단일 엔진 (Method A)

v2.0에서 A/B 비교 실험을 종료하고 **엔진(Method A) 단일 소스**로 확정.

```
사주 입력 → saju_engine.py (v5, deterministic)
         → 100세 타임라인 (세운/월운/대운)
         → score + breakdown 구조
         → DB 저장
```

- 점수는 엔진이 계산 (deterministic, reproducible)
- LLM은 점수를 변경하지 않음

### 4-2. 해설 시스템: 하이브리드 (엔진 점수 + LLM 서사)

```
엔진 score/breakdown + 원시 사주 데이터
         → fortune-prompt.ts (reasoning 지시)
         → Gemini 2.5 Flash
         → "명리 해석가" 톤의 서사 해설
```

- LLM에게 breakdown을 "역추적"하여 원인→영향→조언 서사를 생성하도록 지시
- 점수 숫자의 단순 낭독 금지, 현실적 상황으로 번역
- 해요체(비격식 높임말), 카톡/문자 톤

### 4-3. 기술 스택

| 레이어 | 기술 |
|--------|------|
| 프론트엔드 | Next.js 15, React, Tailwind CSS, Recharts |
| 백엔드 | Next.js API Routes |
| 사주 엔진 | Python (saju_engine.py v5, sajupy 라이브러리) |
| LLM | Google Gemini 2.5 Flash |
| DB | PostgreSQL (Prisma ORM) |
| 인증 | Kakao OAuth + Guest Mode |

## 5. 핵심 기능

### 5-1. 100세 인생 차트

- 출생 ~ 100세까지의 세운(연운) 곡선
- 대운 블록 표시 (10년 단위)
- "올해" 마커 + draw-in 애니메이션
- 기간 선택: 전체 / 10년 / 5년 / 1년(월운)

### 5-2. 보조지표 (기본 OFF, 설정에서 ON)

| 지표 | 설명 |
|------|------|
| 유리한 흐름 | 용신/희신 부합도 기반 긍정 에너지 |
| 변화의 파도 | 합충형파해 기반 변화 강도 |
| 귀인의 도움 | 천을귀인/월덕귀인 등 길신 활성도 |
| 오행 균형도 | 오행 분포 균형 지수 |
| 십성 밸런스 | Radar 차트 (비겁/식상/재성/관성/인성) |
| 이벤트 확률 | 재물/승진/연애/건강 등 카테고리별 이벤트 확률 |

각 지표에 ⓘ 인포메이션 버튼으로 짧은 설명 제공.

### 5-3. 비교하기 (궁합/오버레이)

- 내 사주 목록에서 다른 사람을 선택하여 차트 오버레이
- 겹치는 연도 범위만 표시
- 기간 해설: 두 사람의 관계 궁합/chemistry 분석
  - 이성: 연애/결혼 비중 높임
  - 동성: 업무/우정 비중 높임
- 십성 밸런스: Radar 오버레이
- 이벤트 확률: Clustered bar chart

### 5-4. 운세 해설 (2+5 구조)

평생 해설 1회 생성.

**상단 고정 카드** (항상 펼쳐져 있음):
- 당신의 사주 한 줄
- 인생의 큰 그림

**아코디언** (5개, 각각 다음 구조를 따름):
1. 인생 전반 패턴 (0~100세) — 2~3문단
2. 작년/올해/내년 (3년 맥락) — 원시 재료 → breakdown 원인 → 영향
3. 한 줄 조언 (문자 톤)

아코디언 카테고리:
- 타고난 성격과 잠재력
- 돈과 커리어의 흐름
- 인연과 관계의 지도
- 몸과 에너지의 리듬
- 운명을 바꾸는 습관

### 5-5. 사주 정보 탭

- 사주원국 (천간/지지, 일주 보라색 테두리)
- 오행 분포 (rescaled bar chart + ⓘ 설명)
- 십성 배치
- 대운의 흐름
- 신살/길성 (한글 표시)
- 사주관계 (상세: 묘미반합, 축미충, 축술미삼형 등)

### 5-6. 입력 지원

- 양력/음력 선택
- 음력 선택 시 평달/윤달 구분 UI
- 시간 모름 옵션

## 6. 화면 구조

### 6-1. 온보딩 플로우

```
랜딩 → 로그인(카카오 + 게스트) → 사주 목록 → 사주 입력 → 로딩 → 차트 공개
```

```
[← 뒤로] [개인정보 헤더]

[메인 차트 (세운 곡선)] [⚙️ 지표]

  - 상단 카드 2개 (한줄사주 + 큰그림)
```

## 10. 핵심 원칙

### 원칙 1: 차트가 먼저, 말은 나중에

사용자가 앱을 열었을 때 가장 먼저 보는 것은 차트. 텍스트가 아니라 곡선이 먼저 감정을 건드려야 한다.

### 원칙 2: 복잡함은 기능이 아니라 실패다

기능을 추가할 때마다: "이게 차트를 한 줄 더 보고 싶게 만드는가?" 아니면 넣지 않는다.

### 원칙 3: 우리가 파는 건 정보가 아니라 서사다

차트도, 해설도, 모든 기능도 이 서사를 강화하는 방향이어야 한다.

### 원칙 4: 차트를 다시 보게 만들어라

해설이 차트의 리마인더가 되는 것이 재방문 루프의 핵심.

### 원칙 5: 점수는 FACT, 해설은 REASONING

엔진이 계산한 점수는 사실로 고정. LLM은 그 사실을 "왜 그런지" 서사로 풀어내는 역할.
